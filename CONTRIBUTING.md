# Contributing / 参与贡献

Documentation fixes, clearer examples, and reproducible compatibility reports
are welcome.

欢迎提交文档修正、更清晰的示例和可复现的兼容性报告。

Before opening a pull request:

1. Keep `README.md` and `README.zh-CN.md` consistent.
2. Use `https://api.unifyllm.top` as the canonical API host.
3. Never commit personal credentials or private request data.
4. Make examples read API keys from environment variables.
5. Do not claim unlimited rate, unlimited context, guaranteed latency, or
   uptime without a current verifiable source.
6. Test every link and command changed by the pull request.

提交 Pull Request 前：

1. 保持 `README.md` 与 `README.zh-CN.md` 信息一致。
2. 统一使用 `https://api.unifyllm.top` 作为标准 API 域名。
3. 禁止提交个人凭证或私有请求数据。
4. 示例代码必须从环境变量读取 API Key。
5. 没有当前可验证来源时，不得宣传无限速率、无限上下文、延迟或可用性保证。
6. 检查本次修改涉及的每一个链接和命令。
